## Authentication

Both password reset endpoints are public:

| Endpoint | Gateway `authRequired` | Identity attribute |
| --- | --- | --- |
| `POST /v1/auth/forgot-password` | `none` | `[AllowAnonymous]` |
| `POST /v1/auth/reset-password` | `none` | `[AllowAnonymous]` |

Do not send `Authorization` for these two endpoints. Gateway will still inject its internal `X-Internal-Auth` header when forwarding to Identity; FE/Mobile must not create or send `X-Internal-Auth`.

For normal protected endpoints elsewhere, Gateway validates user access tokens from:

```http
Authorization: Bearer <accessToken>
```

That is not required for the two endpoints in this document.

## Standard Response Envelope

All FE-facing successful object responses are wrapped by `ApiResponse<T>`.

Success shape:

```json
{
  "success": true,
  "statusCode": 200,
  "data": {},
  "meta": {
    "traceId": "req-abc123",
    "timestamp": "2026-07-10T10:00:00.0000000Z"
  }
}
```

Error shape:

```json
{
  "success": false,
  "statusCode": 400,
  "error": {
    "code": "AUTH_OTP_INVALID",
    "message": "Invalid verification code.",
    "fields": [
      {
        "field": "code",
        "message": "The length of 'Code' must be 6 characters. You entered 3 characters."
      }
    ]
  },
  "meta": {
    "traceId": "req-abc123",
    "timestamp": "2026-07-10T10:00:00.0000000Z"
  }
}
```

Notes from code:

- `meta.traceId` comes from `X-Request-Id` when available.
- `meta.timestamp` is UTC ISO-8601 generated by .NET `DateTime.UtcNow.ToString("O")`.
- `error.fields` is only present for validation errors.
- HTTP status line remains authoritative and mirrors body `statusCode`.

## Common Conventions

| Item | Actual behavior |
| --- | --- |
| JSON casing | camelCase in HTTP JSON. C# DTO `OtpTtlMinutes` serializes as `otpTtlMinutes`. |
| Email normalization | Handlers call `Trim().ToLowerInvariant()` before lookup/rate-limit. |
| OTP code | 6 numeric digits, regex `^\d{6}$`, generated with `RandomNumberGenerator.GetInt32(...).ToString("D6")`. |
| OTP purpose | `PASSWORD_RESET`. |
| OTP TTL | 5 minutes. |
| Rate limit key | `identity:pwd_reset_rate:{emailLower}`. |
| Rate limit | max 3 password-reset requests per 1 hour window. |
| Eligible user statuses | only `ACTIVE`. |
| Eligible roles | any `ACTIVE` user with role `PASSENGER`, `DRIVER`, `ASSISTANT`, `OPERATOR_STAFF`, `OPERATOR_ADMIN`, `SYSTEM_ADMIN`. |
| Non-eligible statuses | `PENDING_EMAIL_VERIFICATION`, `PENDING_INITIAL_PASSWORD`, `LOCKED`, `DELETED` do not receive OTP. |
| Google-only account | If user is `ACTIVE` and `PasswordHash = null`, reset is allowed and sets a local password. |
| Refresh token side effect | On successful reset, all active refresh tokens for the user are revoked with reason `PASSWORD_RESET`. |

## Endpoint Overview

| Method | Path | Auth | Description |
| --- | --- | --- | --- |
| POST | `/v1/auth/forgot-password` | Public | Request a password-reset OTP email for an eligible active user. Unknown/non-eligible emails still return generic success. |
| POST | `/v1/auth/reset-password` | Public | Consume a `PASSWORD_RESET` OTP, set a new password, mark OTP used, revoke active refresh tokens. |

## Flow Summary

1. FE calls `POST /v1/auth/forgot-password` with the user's email.
2. API always returns `200` for syntactically valid email unless rate-limited.
3. If the email belongs to an `ACTIVE` user, Identity creates a `PASSWORD_RESET` OTP and queues an `identity.otp.requested` outbox event.
4. User receives a 6-digit OTP by email.
5. FE calls `POST /v1/auth/reset-password` with `email`, `code`, `newPassword`.
6. On success:
   - password hash is replaced,
   - failed login counters are reset by `User.ResetPassword(...)`,
   - OTP is marked used,
   - active refresh tokens are revoked,
   - user can login with the new password.
7. Any old refresh token should be treated as invalid after reset.

## POST `/v1/auth/forgot-password`

### Function Name

Request password reset OTP.

### Method + Full URL

```http
POST https://api.vietride.online/v1/auth/forgot-password
POST http://localhost:3000/v1/auth/forgot-password
```

### Description

Requests a password-reset OTP for an `ACTIVE` user. To avoid account enumeration, unknown emails and non-eligible accounts return the same `200` success shape but do not create/send OTP.

### Headers

| Header | Required | Value | Notes |
| --- | --- | --- | --- |
| `Content-Type` | Yes | `application/json` | Required for JSON body. |
| `Authorization` | No | N/A | Endpoint is public. |
| `X-Request-Id` | No | string | Optional correlation id. Gateway/service can populate `meta.traceId` from it. |

### Path Params

None.

### Query Params

None.

### Request Body

JSON sample:

```json
{
  "email": "user@example.com"
}
```

Field rules from `ForgotPasswordCommandValidator`:

| Field | Type | Required | Validation |
| --- | --- | --- | --- |
| `email` | string | Yes | not empty; valid email address; max length 320 |

### Success Response

HTTP `200 OK`

```json
{
  "success": true,
  "statusCode": 200,
  "data": {
    "email": "user@example.com",
    "otpTtlMinutes": 5
  },
  "meta": {
    "traceId": "req-abc123",
    "timestamp": "2026-07-10T10:00:00.0000000Z"
  }
}
```

Field explanation:

| Field | Type | Meaning |
| --- | --- | --- |
| `success` | boolean | Always `true` for success. |
| `statusCode` | number | Mirrors HTTP status. |
| `data.email` | string | For eligible user: stored normalized user email. For unknown/non-eligible email: lowercased trimmed request email. |
| `data.otpTtlMinutes` | number | OTP validity in minutes; currently `5`. |
| `meta.traceId` | string | Request correlation id. |
| `meta.timestamp` | string | UTC ISO-8601 response timestamp. |

Important account-enumeration behavior:

- Unknown email returns `200`.
- Existing user with status not `ACTIVE` returns `200`.
- Those non-eligible cases do not create OTP/outbox email.


Cause:

## POST `/v1/auth/reset-password`

### Function Name

Reset password with OTP.

### Method + Full URL

```http
POST https://api.vietride.online/v1/auth/reset-password
POST http://localhost:3000/v1/auth/reset-password
```

### Description

Resets the password for an `ACTIVE` user using a `PASSWORD_RESET` OTP. On success, the OTP is marked used and all active refresh tokens owned by that user are revoked with reason `PASSWORD_RESET`.

### Headers

| Header | Required | Value | Notes |
| --- | --- | --- | --- |
| `Content-Type` | Yes | `application/json` | Required for JSON body. |
| `Authorization` | No | N/A | Endpoint is public. |
| `X-Request-Id` | No | string | Optional correlation id. |

### Path Params

None.

### Query Params

None.

### Request Body

JSON sample:

```json
{
  "email": "user@example.com",
  "code": "123456",
  "newPassword": "Password123!"
}
```

Field rules from `ResetPasswordCommandValidator`:

| Field | Type | Required | Validation |
| --- | --- | --- | --- |
| `email` | string | Yes | not empty; valid email address; max length 320 |
| `code` | string | Yes | not empty; exactly 6 characters; regex `^\d{6}$` |
| `newPassword` | string | Yes | not empty; min length 8; max length 128; must contain at least 1 letter; must contain at least 1 digit |

Important: the code does not require a special character for `newPassword`.

### Success Response

HTTP `200 OK`

```json
{
  "success": true,
  "statusCode": 200,
  "data": {
    "userId": "2f3c53b1-a124-4d25-a6d6-8f96bcb255a8",
    "status": "ACTIVE"
  },
  "meta": {
    "traceId": "req-abc123",
    "timestamp": "2026-07-10T10:00:00.0000000Z"
  }
}
```

Field explanation:

| Field | Type | Meaning |
| --- | --- | --- |
| `success` | boolean | Always `true` for success. |
| `statusCode` | number | Mirrors HTTP status. |
| `data.userId` | UUID string | User id whose password was reset. |
| `data.status` | string | User status after reset. Code returns `user.Status.ToString()`, expected `ACTIVE` because only active users can reset. |
| `meta.traceId` | string | Request correlation id. |
| `meta.timestamp` | string | UTC ISO-8601 response timestamp. |

Side effects confirmed in code:

- `User.ResetPassword(...)` replaces `PasswordHash` and resets failed login counters.
- OTP token `used_at` is set.
- All active refresh tokens for `userId` are revoked with reason `PASSWORD_RESET`.

#### 400 - `AUTH_OTP_INVALID`


Confirmed causes:

## FE Integration Notes

### Recommended UI Flow

1. User enters email.
2. Call `POST /v1/auth/forgot-password`.
3. For `200`, always show a generic message such as: "If this email is eligible, an OTP has been sent."
4. Start a 5-minute OTP timer from `data.otpTtlMinutes`.
5. User enters OTP + new password.
6. Call `POST /v1/auth/reset-password`.
7. On success, redirect to login and ask user to login with the new password.

### Error Handling by Code

| HTTP | `error.code` | FE action |
| --- | --- | --- |
| 400 | `AUTH_OTP_INVALID` | Show invalid OTP message; allow retry. |
| 400 | `AUTH_OTP_EXPIRED` | Show expired OTP message; ask user to request a new OTP. |
| 400 | `VALIDATION_ERROR` | Malformed request/body; show field/debug info if useful. |
| 422 | `VALIDATION_ERROR` | Show field-level validation errors from `error.fields`. |
| 429 | `AUTH_OTP_RATE_LIMIT_EXCEEDED` | Disable resend temporarily; ask user to try later. |
| 500 | `INTERNAL_ERROR` | Show generic error; log `meta.traceId`. |
| 502 | `UPSTREAM_UNAVAILABLE` | Show service unavailable; retry later. |

### Token Expiry / Session Handling

After successful reset:

- Existing refresh tokens are revoked in Identity.
- Any old session that tries `POST /v1/auth/refresh` with the old refresh token should fail.
- FE should clear locally stored `accessToken`/`refreshToken` for the reset account and force login again.

### Roles Supported

Password reset is role-agnostic as long as the user status is `ACTIVE`.

Supported roles from code:

```text
PASSENGER
DRIVER
ASSISTANT
OPERATOR_STAFF
OPERATOR_ADMIN
SYSTEM_ADMIN
```

### Statuses That Do Not Receive OTP

These statuses return generic `200` on `forgot-password`, but no OTP is created:

```text
PENDING_EMAIL_VERIFICATION
PENDING_INITIAL_PASSWORD
LOCKED
DELETED
```

On `reset-password`, these statuses return `400 AUTH_OTP_INVALID`.
